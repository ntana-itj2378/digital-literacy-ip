## 1. サイト作成の経緯

### 1-1. 背景と課題
一般の社会人や学生の間で、BitTorrentなどのファイル共有ソフトを「単なるダウンロードツール」と誤認して利用し、無意識のうちに著作権侵害（自動アップロード）を行ってしまうリスクが深刻化しています。既存の警告文だけでは不十分であり、**「実際に起きた事件」と「法的根拠」を結びつけた具体的な啓蒙**が必要となったことがプロジェクトの始まりです。

### 1-2. プロジェクトのミッション
- 抽象的な法律論ではなく、実例（逮捕事例・損害賠償事例）ベースでリスクを自分事化させる。
- 「知らなかった」では済まない法的現実を、視覚的に訴求力のあるデザインで提示する。
- 信頼できるTier1情報源（公的機関・司法判例）に基づく情報の提供。

## 3. サイト作成に関する指摘・合意内容（重要）

これまでの開発プロセスにおいて、ユーザー（USER）より指摘があり合意した主要な事項は以下の通りです。

### 3-1. ブランドトーンと表記
- **装飾の見直し**: プロフェッショナルな印象を保つため、見出しの「【 】」などの過剰な装飾を削除。
- **デザインシステム**: 視覚的なプレミアム感（Glassmorphism / Dark Theme）を全ての新記事に一貫して適用。

### 3-2. サイト構造と保守性
- **ディレクトリの再編**: `/articles/` 直下にあったファイルを `/articles/torrent/` 等にカテゴリ分け。
- **404エラー対策**: 構造変更によるリンク切れ防止のため、`vercel.json` で旧URLから新URLへの301リダイレクトを設定。
- **ビルドシステムの導入**: 個別のHTML編集（ad-hoc）を廃止し、Node.jsとNunjucksテンプレートを用いた一元管理システム（`src/` 配下での管理）へ移行。

### 3-3. 運用の柔軟性
- **手動編集の尊重**: インデックスページのカードタイトルなど、ユーザーが手動で行った独自の調整は上書きせず維持する方針を採用。
- **知財Wikiの定義**: 記事の追加に合わせて随時更新・拡張される「成長型コンテンツ」として定義。

## 4. 今後の方針と将来性

### 4-1. 保守性の確保
現在のビルドエンジン（`build.js`）により、ヘッダーやフッター、共通CSSの変更は1箇所の修正で全ページに反映される堅牢な基盤が整っています。

### 4-2. コンテンツの深化
単なる被害事例の羅列にとどまらず、**「信頼できるリファレンスサイト（知財百科事典）」**としての地位を確立することで、一般ユーザーだけでなく、企業のコンプライアンス研修や教育現場での補助教材としての活用も期待できます。
