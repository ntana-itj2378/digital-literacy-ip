# 知的財産権リスク啓蒙サイト (digital-literacy-ip) 現行仕様書

## 1. プロジェクト概要
一般社会人や学生を対象に、実際の事例をベースとした著作権法違反のリスクを啓蒙するためのウェブサイトです。強烈なインパクトを与えるデザインと、専門性の高い解説を両立した「啓蒙プラットフォーム」として構築されています。

- **プロジェクト名**: digital-literacy-ip
- **コンセプト**: 実例ベースのリスク啓蒙
- **公開URL**: https://digital-literacy-ip.vercel.app/
- **最終更新**: 2026年4月23日

---

## 2. デザイン・UI/UX仕様

### 2-1. デザインコンセプト：警戒モード (Caution Mode)
「Premium High-Contrast Warning Theme」を採用し、法的リスクの重大さを視覚的に伝えます。

- **配色**: 
  - メイン背景：プレミアム・ダークテーマ（黒基調）
  - 本文・サブタイトル：白（#fff）で視認性を最大化
  - 外部リンク：オレンジ（#ff9900）
  - アクセント：イエロー×ブラックの警戒ストライプ（Caution Tape）
- **エフェクト**: パネル外周へのグロー（発光）効果、背景の微細なテクスチャ。
- **フォント**: Noto Sans JP, Outfit。

### 2-2. 画面構成
- **ホーム画面**: 
  - シングルスクリーンのグリッドUI。
  - 「トレント」「ファスト動画」「海賊版」の3つの主要カテゴリを巨大なパネルで選択。
- **記事一覧・詳細**:
  - 「FILE XX」形式の証拠資料を模したカードデザイン。
  - 余計な見出し（「締めの言葉」等）を排除したクリーンな本文構成。
- **Wikiセクション**:
  - 法的知識を体系化するための別セクション。

## 3. 技術・システム構造

### 3-1. 開発スタック
- **テンプレートエンジン**: Nunjucks (custom `build.js` による静的生成)
- **開発ツール/ビルド**: Vite (HMR環境の提供、Environment Injection による環境変数管理)
- **パッケージ管理**: Node.js / npm

### 3-2. フロントエンド機能
- **ナビゲーション**: SPA（Single Page Application）構成。
  - **History API**: ブラウザの「戻る」「進む」操作に完全対応。
  - **URLハッシュ連携**: ハッシュによる直接アクセス管理。
- **記事間遷移**: 記事末尾の「次の記事へ」ボタンの自動パス解決機能。

### 3-3. バックエンド・インフラ仕様
- **アーキテクチャ**: SSG (Static Site Generation)
  - サーバーサイドの動的処理を排し、ビルド時に全てのHTMLを生成することで、極めて高い応答性能とセキュリティを確保。
- **データ管理（ファイルベースDB）**:
  - 従来のRDBMS（SQL等）は使用せず、`src/` 配下の構造化された Markdown およびテキストファイルをデータソースとして利用。
  - 専門データベース（法律・ツール・事件）のデータはフォルダ構造に基づき体系化。
- **インフラ・デプロイ**:
  - **ホスティング**: Vercel
  - **CI/CD**: Git（GitHub/Vercel連携）による自動デプロイ。
  - **環境管理**: 開発・本番環境の差異はビルド時の環境変数注入により吸収。

## 4. コンテンツ構造

### 4-1. 主要カテゴリと啓蒙内容
| カテゴリ | 重点リスク | 解説トピック |
| :--- | :--- | :--- |
| **トレント** | 送信可能化権侵害 | P2P自動アップロード、損害賠償リスク |
| **ファスト動画** | 翻案権侵害 | 著作物価値の毀損、巨額の賠償事例 |
| **海賊版** | 複製権侵害・刑事罰 | ダウンロード違法化、セキュリティリスク |

### 4-2. 知財Wiki (3軸構造)
「法律」「ツール」「サイト・事件」の3セクションで構成。
- **法律**: 著作権法119条（刑事罰）、民法709条（損害賠償）等。
- **ツール/技術**: BitTorrentの仕組み、VPNの匿名性に関する誤解。
- **実例**: 実際の摘発事例や判例。

---

## 5. 保守・拡張性
- **マクロ管理**: Nunjucksマクロにより記事カードやコンポーネントのデザインを一元管理。
- **ディレクトリ構成**:
  - `src/`: ソース
  - `src/articles/`: 記事コンテンツ（カテゴリ別）
  - `src/wiki/`: Wikiポータル
  - `dist/`: 公開成果物
