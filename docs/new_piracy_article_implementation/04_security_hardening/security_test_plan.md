# セキュリティ強化テスト計画 (Security Hardening Test Plan)

本計画書は、「セキュリティ強化実装計画」に基づき導入される対策が正しく機能しているかを検証するための手順と期待結果を定義します。

## 1. 検証項目一覧

| ID | 検証項目 | 検証方法 | 期待される結果 |
| :--- | :--- | :--- | :--- |
| **ST-01** | **HTTPヘッダーの付与** | 本番URLに対し `curl -I` コマンドを実行、またはブラウザのNetworkタブで確認。 | `CSP`, `X-Frame-Options`, `X-Content-Type-Options` 等のヘッダーが正しく含まれていること。 |
| **ST-02** | **CSPの遮断動作** | インラインスクリプトや未許可ドメインのリソース読み込みをテストコードで試行。 | ブラウザのコンソールに「Refused to load... Content Security Policy」の警告が出て遮断されること。 |
| **ST-03** | **ログ出力抑制** | 本番ビルド環境において `logger.log()` および `logger.warn()` を呼び出す。 | コンソールに何も出力されないこと（情報露出の防止）。 |
| **ST-04** | **エラー隠蔽** | 意図的に例外を発生させ、`logger.error()` を実行。 | 本番環境では詳細なスタックトレースが排除され、一般化されたメッセージのみが出力されること。 |
| **ST-05** | **カスタム404ページ** | 存在しないURL（例: `/secret-path`）にアクセス。 | ブランドデザインが適用されたカスタム404ページが表示され、サーバー詳細が露出しないこと。 |

## 2. 具体的なテスト手順

### ST-01: セキュリティヘッダー確認
1.  ターミナルで `curl -I https://digital-literacy-ip.vercel.app/` を実行。
2.  レスポンスヘッダー内の以下を確認：
    *   `content-security-policy`
    *   `x-frame-options: DENY`
    *   `x-content-type-options: nosniff`

### ST-02: CSP遮断テスト
1.  一時的に外部ドメイン（例: `https://evil.com/script.js`）を読み込むタグを追加してビルド。
2.  ブラウザでアクセスし、読み込みが拒否されることを確認。

### ST-03 & ST-04: ロガーテスト
1.  `logger.log("Internal Data")` を実行するコードを配置。
2.  ビルドを行い、ブラウザコンソールを確認して "Internal Data" が表示されないことを確認。

### ST-05: 404ページ確認
1.  本番URLに `/invalid` を付けてアクセス。
2.  「Page Not Found」などのカスタムUIが表示されることを確認。

## 3. 合否判定基準
*   **合格**: 全ての検証項目（ST-01 〜 ST-05）で期待結果が得られること。
*   **不合格**: いずれかのヘッダーが欠落している、または本番環境でデバッグログが露出している場合。

---
**💡 運用上の注意**
本番環境でのテストは、デプロイ完了直後に実施します。万が一CSPの設定により正当なリソース（Google Fonts等）が遮断された場合は、即座に計画書を修正し、ポリシーを調整します。
